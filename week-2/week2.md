# Week 2 — Competency 2 (Code literacy and documentation)

My companion files `quote_word_stats.md` and `demo_word_count.md` explain each of the Week 2 scripts in plain language: what it is for, what files it depends on, how to run it from the terminal, and what output to expect—so a non-technical reader can use the repo without reading Python first. In `demo_word_count.py`, the `count_words()` docstring states the argument type, that it splits on whitespace to compute a count, what it returns, and why the function exists (to compare response lengths across participants). In `quote_word_stats.py`, I use inline comments to separate phases—resolving the CSV path, building the per-author counts, and printing the summary—so the control flow reads as intentional steps rather than a single opaque block. I am also practicing Git messages that point to a concrete change (for example, noting when comments were added to a specific file) rather than vague one-word updates, so history stays legible as the project grows.
